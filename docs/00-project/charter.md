# Project Charter

## Project Name

JJ Steak Digital Bookkeeping

## Problem Statement

JJ Steak needs a centralized backoffice system to replace manual Excel-based recording for multi-outlet finance, inventory, and accounting workflows. Current manual process is slow, error-prone, difficult to audit, and hard to consolidate across outlets.

## Primary Goals

1. Generate accurate Laba Rugi per outlet and consolidated.
2. Generate accurate Neraca per outlet and consolidated.
3. Provide supporting cashflow analysis.
4. Provide supporting inventory analysis.
5. Reduce manual reconciliation workload.
6. Improve approval control and auditability.

## Scope In

1. Backoffice web application.
2. Master data.
3. Finance: daily sales summary, bank book, payment request, approvals, AP/AR reports.
4. Logistics: PO, receiving, stock, usage, opname, adjustment, recipe.
5. Accounting: journal, ledger, trial balance, Laba Rugi, Neraca, closing.
6. Reporting and analytics.

## Scope Out

1. POS replacement.
2. Mobile app.
3. Payroll.
4. Fixed assets.
5. Full tax modules such as e-Faktur/e-Bupot.
6. Direct bank API integration unless added later.

## Success Metrics

1. Laba Rugi and Neraca generated from system without manual spreadsheet compilation.
2. Daily sales and bank movement reconciled per outlet.
3. Stock position and valuation available per outlet.
4. All approvals and financial changes traceable in audit log.
5. Month-end close follows system checklist.
