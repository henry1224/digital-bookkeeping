# Module PRD: Master Data

## Goal

Provide consistent reference data for all finance, logistics, and accounting workflows.

## Features

1. Item Master
2. Group Item
3. Supplier
4. Customer
5. Bank Account
6. Chart of Account
7. Outlet
8. User & Role
9. Unit of Measure
10. Tax Setting
11. Audit Log view

## Acceptance Criteria

1. User can create/edit/deactivate master data based on permission.
2. Deactivated master data cannot be used in new transactions.
3. Existing transactions keep old references.
4. All changes are audit logged.
5. Import/export available for Item, Supplier, COA.
